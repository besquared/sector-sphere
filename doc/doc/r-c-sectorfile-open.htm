Reference: Classes: SectorFile

open

The open function opens a Sector file.

int open
(
   const string& filename,
   int mode = SF_MODE::READ
   const string& hint = ""
);
Parameters
filename
[in] the name of the file to be opened.
mode
[in] file open mode, default is READ ONLY. The open mode is defined as constants in structure SF_MODE. The list of defined file open modes can be found in the table below:
Mode Comment
SF_MODE::READ read only
SF_MODE::WRITE write only
SF_MODE::RW read and write, equivelant to SF_MODE::READ | SF_MODE::WRITE
SF_MODE::TRUNC the file is emptied upon opening, must be used togeter with WRITE mode. NOT SUPPORTED YET.
SF_MODE::APPEND the file write position offset is set to end of file upon opening, must be used together with WRITE mode.
SF_MODE::SECURE file IO is in secure mode, must be used togeter with one of the READ/WRITE mode.
SF_MODE::HiRELIABLE data update is replicated at real time if this option is set; otherwise, replica is periodically checked .
hint
[in] hint is an IP address that Sector may use to choose an IO node. Sector will choose the nearest node to hint, including hint itself. By default, Sector will choose nearest node to the client computer.
Return Value

On success, the function returns 0. If error occurs, a negative value is returned.

Error Name Error Code Comment
     
Description

File open mode can be a combined value of multiple options, e.g., SF_MODE::READ | SF_MODE::WRITE.

When necessary, Sector file IO can be encrypted by specifying SF_MODE::SECURE in the mode parameter.
