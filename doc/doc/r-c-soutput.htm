SOutput

The SOutput class is used to store the processing result of User Defined Functions (UDF).

Class Members
Member Type Description
m_pcResult char* The buffer to store the processing result
m_iBufSize int The physical size of the buffer m_pcResult
m_iResSize int Size of data stored in m_pcResult
m_pllIndex int64_t* Offset index of data stored in m_pcResult
m_iIndSize int The physical size (number of 64bit integers) of m_pllIndex
m_iRows int Number of rows/records of the result
m_piBucketID int* The bucket ID (hash value) of each record
m_llOffset int64_t If the input is a file name and there is no enough buffer to hold the result, the current processing may stop in the middle of the file. This parameter is used to store the current position, and the next call of the UDF will continue from this position. The value of this variable is 0 for the first call, and should be set to -1 when all data is processed.
m_strError string The text error message that can be sent back to the client application.
