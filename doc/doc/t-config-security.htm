Configuring the Security Server

The security server maintains an access control list for the master/slave nodes and all the uers information. The access control list decides which node can join the Sector system as a master or slave node. The user information includes user account, password, access IP control, and file IO permission.

master_acl.conf

The "master_acl.conf" file includes a list of IP addresses (e.g., 192.168.0.1) or IP address ranges (e.g., 192.168.0.0/24) from which a computer can join the Sector system as a master server. Each line represents only one address or one address range. The address range is represented by bit mask. That is, 192.168.0.0/24 means 192.168.0.0 - 192.168.0.255. By default, any node (0.0.0.0/0) can join the system as a master.

#list of IP addresses or ranges from which a slave can join the system
#by default the system is open: i.e., any node can become a slave node
#you should change this for a production system
#E.g., 192.168.0.0/24 allows all IPs within 192.168.0.0-255 to join

0.0.0.0/0

slave_acl.conf

The slave access control list has exactly the same format as master_acl.conf, except that its information is stored in slave_acl.conf.

User Accounts

The user account information is stored in ./codeblue2/conf/users. Information about each user is stored in one text file. For example, information of user "test" is stored in ./codeblue2/conf/users/test.

Currently the account informtion is only stored in clear text. We plan to encrypt them in the first production quality release. The "PASSWORD" section is the user's password in clear text. The "READ_PERMISSION" lists files or directories that the user can read, one per line. Similarily, the "WRITE_PERMISSION" is a list of files or directories that the user can write. The "EXEC_PERMISSION" is a TRUE or FALSE value that indicates if the user can run Sphere applications. The "ACL" lists IP addresses or IP address ranges from which the user can login, one per line. Finally, a quota of disk space usage can be applied to a user. The quota includes the total size of the writable files or directories of the user, without counting the replicas.

In the confiuguration file, the option value must be preceded by a "TAB" charactor. The "#" charactor can be used for comment lines.

#user's password
PASSWORD
    xxx

#directories that the user can read, one per line
#default is none, which means the user cannot read anything, same for write
READ_PERMISSION
    /test
    /angle
    /

#directories that the user can write or modify
WRITE_PERMISSION
    /angle
    /

#if the user is allowed to run Sphere process, default is false.
#All capital letters "TRUE" must be used, otherwise it will be treated as false
EXEC_PERMISSION
    TRUE

#list of IP addresses from which the user is allowed to login, one per line.
#this field must exist, otherwise the user will not be able to login
ACL
    0.0.0.0/0

#the user's quota, which is the size limit of all the directories he/she has the write permission
#unit is MB
QUOTA
    1000000
