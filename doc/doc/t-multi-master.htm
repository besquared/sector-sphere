Installation

Starting more Master Servers

For better availability and to serve more clients, more master servers can be started. A new master server can be started at any time. However, it is recommended that the new server is started when no client connects to the system, otherwise client requests may experience short delay due to the reorganization of the Sector masters.

master.conf

The configuration file of the new master must be consistent to those existing masters. In particular, the number of replicas should be the same as other masters. It is recommended to have exactly one security server in the system. However, a master server may specify a different security server if necessary. In this case, the configuration files of the security servers must be the same.

topology.conf

Copy the "topology.conf" file from an existing master server to the ./conf directory of the new master server, if there is one.

slaves.list

Copy the "slaves.list" file from an existing master server to the ./conf directory of the new master server, if there is one.

master_node.key/cert

Copy the master key/certificate files of an existing master server to the ./conf directory of the new master server. Do NOT generate a new set of key/certificate pair for the new master.

Now, to start the new master, simply run ./start_master IP port, where IP and port is the address of an existing master server.
