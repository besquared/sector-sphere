Configuring the Client

The configuration file "client.conf" is not required to run the client. However, without client.conf, you will be prompted to provide master server information and user account information every time you run a Sector command. This information can be stored in the client configuration file. The file client "client.conf" is located at ./codeblue2/conf.

client.conf

A copy of the client configuration file can be found below. The MASTER_ADDRESS is the address of the master server in the format of <ip:port> or <domain_name:port>. The "USERNAME" and "PASSWORD" options specify the user's account name and password. Finally, the CERTIFICATE option stores the location of the master's certificate, which is used to verify the identity of the master node.

In the confiuguration file, the option value must be preceded by a "TAB" charactor. The "#" charactor can be used for comment lines.

Note that in order to allow Sector-FUSE work properly, the "CERTIFICATE" path must be an abosolute path.

#Master address
MASTER_ADDRESS
    ncdm161.lac.uic.edu:6000

#user's name
USERNAME
    text

#password
PASSWORD
    xxx

#master's certificate file
CERTIFICATE
    /home/gu/master_node.cert
