Installation

Mounting Sector as a User Space File System

Sector includes a mountable user space file system interface implemented using FUSE. This allows Sector to be mounted and used just like a directory on the local file system.

Sector-FUSE is located in ./sector-sphere/fuse. By default, this directory is not compiled together with rest of the system. To use this feature, install FUSE (version 2.6 or above) first, if it is not installed previously, then compile ./sector-sphere/fuse, and run sector-fuse to mount the Sector file system.

NOTE: Because Sector is a network file system and FUSE also incurs additional round trip messages, using Sector-FUSE is inefficient for small IO .

Install FUSE

FUSE has been officially merged into the Linux kernel tree since kernel version 2.6. To install FUSE, you may simply use the FUSE package for the specific Linux distribution. For example, on Debian, you can use:

#apt-get install libfuse-dev
#apt-get install libfuse2
#apt-get install fuse-utils

Note that you need root privilege to run these commands. After this, FUSE will be installed and started. A "fuse" group will be created and you need to put all Sector-FUSE users into this group. You may also start Sector-FUSE with "root". However, this is not recommended.

You may also download FUSE source code and install from the source. For more information about FUSE, please visit http://fuse.sourceforge.net.

Compile and Start Sector-FUSE

In ./sector-sphere/fuse, run "make", which will compile the Sector-FUSE utility. Note that FUSE must be installed and started and the Sector system must be successfully compiled and the client configuration file must exist in ./sector-sphere/conf. After a successful make, run ./sector-fuse <local dir>, where <local dir> is an empty directory on the local file system. Now you can access the Sector file system via this directory.

To unmount Sector-FUSE, run "fusermount -u <local dir>".

gu@ncdm90:~/sector-sphere/fuse$ mkdir test
gu@ncdm90:~/sector-sphere/fuse$ ./sector-fuse test
gu@ncdm90:~/sector-sphere/fuse$ cd test
gu@ncdm90:~/sector-sphere/fuse/test$ ls
bar sector2 test.secure5.txt
bar2 sector2.idx test.secure6.txt
fraudcount sector8 uofc2-1185190800-dump.7.3131.pcap
gu@ncdm90:~/sector-sphere/fuse/test$ cd ..
gu@ncdm90:~/sector-sphere/fuse$
gu@ncdm90:~/sector-sphere/fuse$ fusermount -u test
