Configuring the Slave

There is only one configuration file "slave.conf" under ./sector-sphere/conf. This file must be configured before the slaves can be started.

slave.conf

A copy of the slave configuration file can be found below. The MASTER_ADDRESS is the address of the master server in the format of <ip:port> or <domain_name:port>. The "DATA_DIRECTORY" option is the directory that Sector uses to store data files and system information. If the directory does not exist, Sector will create it automatically, provided that the slave program has write permission to the directory. There are more options in the configuration file and the explanation for each option is also included immediately above the option name.

In the configuration file, the option value must be preceded by a "TAB" character. The "#" character can be used for comment lines.

#Master address
MASTER_ADDRESS
    ncdm161.lac.uic.edu:6000

#Data directory
DATA_DIRECTORY
    /home/u2/yunhong/work/data/

#Maximum data size, in MB
#Optional: use this if you do not want Sector to use all available disk space
#MAX_DATA_SIZE
#    1000000

If the data directory already contains local files before the node joins the system, these files will become part of the Sector file system, except those with name conflicts, which will be moved into DATA_DIRECTORY/.attic.
