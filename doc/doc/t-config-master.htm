Installation

Configuring the Master Server

There are three configuration files for the master server: master.conf, topology.conf, and slaves.list. These files are located in the configuration file directory (./codeblue2/conf). The file "master.conf" includes the basic information required to start the master server, such as server port number. This file MUST be configured. The file "topology.conf" specifies the topology information of the slaves network. This is optional. The file "slaves.list" includes a list of slave nodes that the master can automatically start or shutdown. This is also optional.

master.conf

A copy of the master configuration file can be found below. The SECTOR_PORT is the port number on which the Sector master will listen to requests from clients. Note that although you only specifily one port number here, Sector will actually use two consectutive numbers. For example, if you use port 6000, Sector will use both ports 6000 and 6001. The SECURITY_SERVER is the address of the security server in the format of <ip:port> or <domain_name:port>. The option "REPLICA_NUM" specifies the number of replicas of each file in the system. Finally, you also need to specify a directory to which Sector can store temporary files under the option "DATA_DIRECTORY". If the directory does not exist, Sector will create it automatically, provided that the master program has write permission to the directory. There are more options in the configuration file and the explaination for each option is also included immediately above the option name.

In the confiuguration file, the option value must be preceded by a "TAB" charactor. The "#" charactor can be used for comment lines.

#SECTOR server port number
SECTOR_PORT
    6000

#security server address
SECURITY_SERVER
    ncdm161.lac.uic.edu:5000

#number of replicas of each file
REPLICA_NUM
    2

#maximum active users
MAX_ACTIVE_USER
    128

#data directory, for the master to store temporary system data
#this is different from the slave data directory and will not be used to store data files
DATA_DIRECTORY
    /home/u2/yunhong/work/data/

#slave node timeout, in seconds
#if the slave does not send response within the time specified here,
#it will be removed and the master will try to restart it
#SLAVE_TIMEOUT
#    60

topology.conf

Sector assumes that ther slave nodes are computers in one or more clusters, while clusters can be allocated in one or more data centers, and so on. This is a multi-level hierarchical topology. The position of each IP range is represented by "/L1/L2/.../Ln" is an n-level structure and L1 is the first level. For example, in a 2-level topology, "/1/2" means the 2nd cluster in the 1st data center. The number of levels for each IP address or IP address range must be equal.

The slave IPs can be either a single IP address, e.g., 192.168.1.1, or a group of continuous IPs represented by IP mask, e.g., 192.168.0.0/24 (means 192.168.0.0 - 192.168.0.255) .

For those slaves whose IPs are not in this list, they will be put into "/0/0/.../0" by default. Therefore you should avoid using "/0/0/.../0" in this list. Also, the level ID does not need to be continuous.

#The topology is a hierachical structure. You may use single IP or IP range for the address.
# There may be multiple levels in the structure (e.g, data center and rack in 2-level structure)
# Only digit is allowed to notate topology, starting with "/".
# All digits should be greate than 0. Do not use 0 and negative values, which are reserved for internal use.
# All topology paths must have the same number of levels......

192.168.136.0/26     /1/1
206.220.241.0/24     /1/2
67.58.56.64/26       /2/1
192.168.136.64/26    /3/1
131.193.181.0/24     /4/1

slaves.list

When using "start_all" and "stop_all" to start and stop all slaves automatically from the master node, the list of slave information must be specified in the "slaves.list" configuration file. In this text file, each line represents one slave in the following format:

<user name>@<ip address> <location of the sector slave program on the node>

gu@192.168.136.8 /home/generichome/gu/sector-test/slave/
gu@192.168.136.9 /home/generichome/gu/sector-test/slave/
gu@192.168.136.13 /home/generichome/gu/sector-test/slave
gu@192.168.136.14 /home/generichome/gu/sector-test/slave

gu@192.168.136.16 /home/generichome/gu/sector-test/slave/
